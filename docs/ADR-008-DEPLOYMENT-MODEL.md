# ADR-008: نموذج النشر والتسليم — PWA على Vercel بدون اشتراكات

**الحالة:** مقبول  
**التاريخ:** 2026-06-24  
**المقرِّر:** محمد مسعود  

---

## السياق

المنتج (LenaBeauty) جاهز للبيع لعملاء صالونات التجميل.  
نمط البيع المستهدف: **رسوم لمرة واحدة** — المالك يدفع مرة ويحصل على نظام يشتغل إلى الأبد بدون اشتراكات شهرية.

متطلبات المالك:
- يفتح التطبيق من موبايله (iPhone/Android) زي أي تطبيق
- يفتحه من الكمبيوتر في المكتب بدون تثبيت
- البيانات تبقى محفوظة (سحابية)
- ما يحتاج خبرة تقنية للتشغيل اليومي

قيود على الحل:
- التطبيق أصلاً Vite + React (SPA)
- البيانات في Supabase
- لا يوجد backend مخصص
- يجب أن تكون التكلفة المستمرة قريبة من الصفر

---

## الخيارات المدروسة

### الخيار A — PWA على Vercel + Supabase (المختار)

| البُعد | التقييم |
|--------|---------|
| التعقيد | منخفض |
| التكلفة للمطور | صفر (Vercel free, Supabase free) |
| التكلفة للعميل | اختياري: Supabase Pro ($25/شهر) لو الداتا تجاوزت 500MB |
| التثبيت على الموبايل | "إضافة إلى الشاشة الرئيسية" — بدون App Store |
| الصيانة | Vercel يتولى الـ hosting، Supabase يتولى الـ DB |
| العمل Offline | جزئي (الـ shell يعمل، البيانات تحتاج إنترنت) |
| التوافق | iOS 16.4+، Android Chrome، Desktop Chrome/Edge/Safari |

**مميزات:**
- صفر تعقيد في التثبيت للمالك — رابط واحد ويضغط "إضافة"
- التطبيق عندك أصلاً PWA-ready (manifest.json + sw.js موجودين)
- إمكانية التحديث المركزي: تحدّث الكود مرة واحدة، كل العملاء يأخذون التحديث تلقائياً
- لا تكاليف App Store ($99/سنة Apple, $25 مرة واحدة Google)
- يعمل على أي جهاز بمتصفح حديث

**عيوب:**
- لا يعمل في غياب الإنترنت (البيانات في السحابة)
- iOS لا تدعم Push Notifications كاملاً في PWA
- المظهر في قائمة التطبيقات أبسط من native app

---

### الخيار B — Tauri (تطبيق Desktop) + PWA للموبايل

| البُعد | التقييم |
|--------|---------|
| التعقيد | عالي |
| التكلفة للمطور | وقت تطوير إضافي (2-4 أسابيع) |
| التثبيت | يحتاج تثبيت .exe / .dmg |
| الصيانة | كل تحديث يحتاج ملف جديد للعميل |

**حكم:** مبالغة في الوضع الحالي. الكود يدعم Tauri مستقبلاً (`src/infrastructure/tauri/`) لكن ليس ضرورياً الآن.

---

### الخيار C — React Native / Expo

| البُعد | التقييم |
|--------|---------|
| التعقيد | عالي جداً (rewrite كامل) |
| التكلفة | تطوير ضخم + App Store fees |
| الفائدة | تجربة native أفضل |

**حكم:** مستبعد كلياً. التطبيق موجود ويعمل، لا مبرر لإعادة الكتابة.

## القرار

**الخيار A — PWA على Vercel + Supabase.**

كل عميل يحصل على:
1. **Supabase project منفصل** (بيانات معزولة بالكامل بين العملاء)
2. **Vercel deployment منفصل** — أو deployment واحد مع `VITE_CENTER_ID` مخصص لكل عميل
3. **رابط مخصص** مثل `salon-lena.vercel.app`
4. **دليل تثبيت** بصور توضيحية للمالك

---

## التأثيرات

**ما يصبح أسهل:**
- تسليم عميل جديد في ساعة واحدة (لا تطوير إضافي)
- تحديثات الكود تصل لجميع العملاء فوراً
- لا دعم فني لمشاكل التثبيت

**ما يجب مراعاته:**
- كل عميل = Supabase project مستقل (لعزل البيانات)
- لو الداتا تجاوزت 500MB على Supabase Free: العميل يرقّي لـ Pro ($25/شهر) أو تنقل DB لـ Railway/Fly.io
- النسخ الاحتياطية: يجب تعليم العميل على زر "Export" أسبوعياً

---

## خطة التنفيذ لكل عميل جديد

```
1. إنشاء Supabase project جديد (10 دقائق)
2. تشغيل SUPABASE_BASE_SCHEMA_BOOTSTRAP.sql
3. إنشاء admin user في Supabase Auth
4. إنشاء Vercel deployment جديد بـ ENV vars خاصة بالعميل
5. تسليم العميل: رابط + دليل التثبيت (هذا الملف)
```

---

*هذا القرار يُراجَع لو:*
- *عدد العملاء تجاوز 20 (يصبح multi-tenant أجدى)*
- *طُلب العمل Offline بشكل كامل (يصبح Tauri مناسباً)*
- *طُلب Push Notifications على iOS (يحتاج native app)*
